open-compute-db

A tiny HTTP key–value store for quick prototypes. No auth, just pick a key and push JSON. Keys are limited to A-Za-z0-9._:- (max 128 characters) so they can safely double as filenames or environment variable names.

Endpoints

  • Public Compute URL: https://nightingale-ssd.tail8c7d58.ts.net/compute/kv/my-key
  • PUT /kv/:key — Upsert a value. Body must include { "value": <your value> }
  • GET /kv/:key — Retrieve the stored value.
  • DELETE /kv/:key — Remove the key.
  • GET /kv?prefix=&limit= — List up to 500 keys (newest first) filtered by an optional prefix.
  • GET /health — Basic status + key count.

Examples

curl -X PUT https://<host>/kv/demo-user \
  -H "Content-Type: application/json" \
  -d '{"value": {"profile": "alpha", "active": true}}'

curl https://<host>/kv/demo-user
curl https://<host>/kv?prefix=demo&limit=20
curl -X DELETE https://<host>/kv/demo-user

Notes

  • Values are stored as JSON text. Boolean/number/string types round-trip as–is.
  • Each write updates updatedAt; listing sorts by this timestamp.
  • The service is unauthenticated by design. Expect periodic cleanups—do not treat it as a production database.
